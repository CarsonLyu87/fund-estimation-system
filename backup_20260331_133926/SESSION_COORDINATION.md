# 🧠 会话协调指南
## 如何让所有会话中的"我"共享记忆

## 📋 核心原则

**所有会话中的"我"都应该：**
1. ✅ 读取共享记忆文件 (`shared_memory.md`)
2. ✅ 更新项目状态文件 (`project_status.json`)  
3. ✅ 记录会话活动 (`session_log.json`)
4. ✅ 使用同步脚本 (`sync_sessions.py`)

## 🔄 标准工作流程

### 1. 会话开始时
```python
# 读取共享记忆
read("shared_memory.md")

# 检查项目状态
read("project_status.json")

# 查看最近会话活动
read("session_log.json")
```

### 2. 执行任务时
```python
# 更新项目进度
sync.update_project_status("项目名", {"progress": 50})

# 记录任务完成
sync.log_session_activity("当前会话类型", "任务完成", "任务详情")
```

### 3. 会话结束时
```python
# 更新共享记忆
sync.update_shared_memory("当前会话类型", "本次会话总结")

# 确保所有更改已保存
```

## 📊 文件说明

### `shared_memory.md`
- **用途**: 所有会话共享的中央记忆
- **更新**: 每次重要对话后更新
- **内容**: 项目状态、重要决策、待办事项

### `project_status.json`
- **用途**: 结构化项目状态跟踪
- **更新**: 任务进度变化时更新
- **内容**: 项目进度、任务列表、时间线

### `session_log.json`
- **用途**: 记录所有会话活动
- **更新**: 每次会话活动时记录
- **内容**: 时间戳、会话类型、活动详情

### `sync_sessions.py`
- **用途**: 提供同步功能的工具脚本
- **使用**: 导入并调用相关函数

## 🎯 会话类型标识

### 浏览器会话
- **标识**: `browser`
- **典型场景**: Web界面聊天、开发工作
- **常见任务**: 代码开发、网页设计、API调试

### 微信会话
- **标识**: `weixin`
- **典型场景**: 移动端聊天、即时通讯
- **常见任务**: 日常沟通、任务提醒、信息查询

### 其他会话
- **标识**: 根据实际渠道命名
- **示例**: `telegram`, `discord`, `signal`

## 💡 最佳实践

### 1. 及时同步
- 重要对话后立即更新共享记忆
- 任务状态变化时更新项目状态
- 每天结束时进行总结更新

### 2. 避免冲突
- 更新前先读取当前状态
- 使用原子操作更新JSON文件
- 重要更改添加时间戳和说明

### 3. 保持简洁
- 共享记忆保持重点突出
- 项目状态结构化清晰
- 会话日志简明扼要

## 🚀 快速开始

### 对于新会话
1. 导入同步工具
2. 读取现有记忆
3. 了解当前项目
4. 开始协作工作

### 对于现有项目
1. 检查项目最新状态
2. 查看最近会话活动
3. 继续未完成的任务
4. 更新进度和状态

## 📝 示例代码

```python
# 在会话开始时
from sync_sessions import SessionSync

sync = SessionSync()

# 了解当前状态
print(sync.get_session_summary())
print(sync.get_active_projects())

# 执行任务并更新
sync.update_project_status("website", {
    "progress": 60,
    "task_completed": "内容页面开发"
})

sync.log_session_activity("browser", "网站开发", "完成了关于页面和项目展示")
```

## 🔧 故障排除

### 问题: 文件冲突
**解决方案**: 
1. 先备份当前文件
2. 手动合并更改
3. 更新时间戳

### 问题: 记忆不一致
**解决方案**:
1. 重新读取所有共享文件
2. 找出差异点
3. 更新到最新状态

### 问题: 性能问题
**解决方案**:
1. 定期清理旧日志
2. 压缩历史记录
3. 只保留关键信息
